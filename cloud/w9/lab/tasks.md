# Lab Tasks

## Priority A - Required
- [ ] Identify which W8 manifests or workloads will enter GitOps first.
- [ ] Draft the ArgoCD application structure.
- [ ] Identify the first metrics to scrape for the chosen application.
- [ ] Identify the first logs to centralize.
- [ ] Draft a dashboard with request rate, error rate, and p95 latency.
- [ ] Choose one application or service for canary rollout.
- [ ] Draft the canary sequence and abort rule.
- [ ] Update `evidence.md` as work progresses.

## Priority B - Strongly recommended
- [ ] Write a commit-to-production flow in plain text.
- [ ] Note the minimum blockers before this can run for real.
- [ ] Prepare a short show-and-tell demo script.
- [ ] List risks or assumptions clearly.

## Definition of done
- [ ] `scope.md` is refined for the actual W8 platform.
- [ ] `deliverables.md` is filled in with concrete outputs.
- [ ] `evidence.md` contains proof of work and demo evidence.
